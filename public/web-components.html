HTTP/2, SPDY and QUIC rulez and here is the evidence

The purpose of this site is to show what is the difference when same content is being served via different protocols - HTTP, HTTP/2, SPDY and raw SSL.

The current page uses a few Web Components. Web Components are still heavy beasts, so, they are perfect candidate to see if they will load faster via HTTP 2.0 and SPDY.

View current page via some of the following protocols:

HTTP HTTP/2 SPDY HTTPS

Current browser support for HTTP/2

  •  Chrome Canary - you have to enable SPDY/4 flag by typing "chrome://flags" in address bar and search for "HTTP/2" or "SPDY/4"
  •  Firefox
  •  Internet Explorer (in development)
  •  Safari

Webpagetest.org is our best friend in performance checking!

Using Webpagetest.org to check how the site behaves is highly recommendable. The development tools of browsers are very good starting point too. When you test, be sure that you checked via mobile networks and slow Internet connections too.
