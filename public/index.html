HTTP/2, SPDY and QUIC rulez and here is the evidence

The purpose of this site is to show what is the difference when same content is being served via different protocols - HTTP, HTTP/2, SPDY and raw SSL.

The current page is a typical online shop page, where different products are listed. It contains a few CSS files, JavaScript files and images. All of them were not joined, neither sprites were created intentionally.

View current page via some of the following protocols:

HTTP HTTP/2 SPDY HTTPS

Note: Chrome stable currently does not support HTTP/2. Use Canary instead with SPDY/4 (HTTP 2.0) flag enabled. You may enable it from the flags page - type "chrome://flags" in address bar and search for "HTTP/2" or "SPDY/4".

Click on the links above to feel what is difference on site loading, use webpagetest.org to check the site, or use the development tools of Chrome, Firefox, Safari or IE11. You may also try via mobile browsers and also mobile networks.
